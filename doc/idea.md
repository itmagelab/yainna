# Идея проекта: Лендинг салона красоты "Я Инна"

## Описание

Проект представляет собой **сайт-визитку** для салона красоты "Я Инна" в городе Сочи. Сайт написан на Rust с использованием фреймворка Yew и компилируется в WebAssembly для работы в браузере.

## Основная цель

Создание современной и привлекательной онлайн-визитки салона красоты "Я Инна" с основной информацией о салоне, услугах и контактах для привлечения клиентов в Сочи.

## О салоне "Я Инна"

Салон "Я Инна" — это профессиональный салон визажа и ухода за бровями и ресницами в Сочи, специализирующийся на:

- Профессиональный визаж (дневной, вечерний, свадебный макияж)
- Уход за бровями (коррекция, окрашивание, ламинирование)
- Уход за ресницами (наращивание, ламинирование, окрашивание)

## Структура сайта

Сайт состоит из 6 основных блоков, расположенных последовательно на одной странице:

### 1. Hero Section (Заголовок и призыв к действию)

**Назначение**: первое впечатление, привлечение внимания и побуждение к действию

**Содержание**:
- Заголовок: "Студия красоты"
- Слоган: "Привет, я Инна"
- Краткое описание (1-2 предложения о салоне)
- Кнопки призыва к действию:
  - "Записаться онлайн" — ссылка на Telegram/WhatsApp
  - "Позвонить" — прямая ссылка на телефон
  - "Смотреть услуги" — якорь к разделу услуг

**Дизайн**:
- Полноэкранный блок с привлекательным фоном
- Градиент или фотография салона
- Центрированный текст
- Выделяющиеся кнопки

### 2. Обо мне (About Section)

**Назначение**: представление мастера, создание доверия

**Содержание**:
- Фотография Инны (профессиональное фото)
- Заголовок: "Обо мне" или "Познакомьтесь с мастером"
- Текст-описание:
  - Опыт работы
  - Специализация
  - Образование и сертификаты
  - Философия работы
  - Почему клиенты выбирают именно этот салон
- Достижения и награды (если есть)

**Дизайн**:
- Двухколоночная раскладка: фото + текст
- Светлый фон для легкости восприятия
- Акцентные элементы (иконки, цитаты)

### 3. Услуги и цены (Services Section)

**Назначение**: детальная информация о предоставляемых услугах

**Содержание**:
- Заголовок: "Наши услуги"
- Три категории услуг в виде карточек:

**Визаж**:
- Дневной макияж — описание, время, цена
- Вечерний макияж — описание, время, цена
- Свадебный макияж — описание, время, цена
- Макияж для фотосессии — описание, время, цена

**Брови**:
- Коррекция бровей — описание, время, цена
- Окрашивание бровей — описание, время, цена
- Ламинирование бровей — описание, время, цена
- Архитектура бровей — описание, время, цена

**Ресницы**:
- Наращивание ресниц — описание, время, цена
- Ламинирование ресниц — описание, время, цена
- Окрашивание ресниц — описание, время, цена
- Снятие наращенных ресниц — описание, время, цена

**Дизайн**:
- Карточки с иконками
- Сетка 3 колонки (на мобильных — 1 колонка)
- Hover-эффекты для интерактивности
- Кнопка "Записаться" под каждой услугой

### 4. Галерея работ (Portfolio Section)

**Назначение**: демонстрация качества работы, визуальное подтверждение навыков

**Содержание**:
- Заголовок: "Наши работы" или "Портфолио"
- Подзаголовок: "Фотографии до и после"
- Сетка фотографий с работами:
  - Примеры макияжа
  - Оформление бровей
  - Работы с ресницами
  - Фото довольных клиентов (с разрешением)

**Дизайн**:
- Красивые рамки вокруг фотографий
- Масонри-сетка или равномерная сетка 3-4 колонки
- Увеличение фото при клике (модальное окно)
- Фильтры по категориям (Визаж / Брови / Ресницы / Все)
- Эффект hover с плавным появлением описания

### 5. Отзывы клиентов (Testimonials Section)

**Назначение**: социальное доказательство, повышение доверия

**Содержание**:
- Заголовок: "Отзывы наших клиентов"
- Карточки с отзывами, содержащие:
  - Фото клиента (аватар) или инициалы
  - Имя клиента
  - Рейтинг (5 звезд)
  - Текст отзыва
  - Дата отзыва
- 6-9 отзывов с возможностью прокрутки
- Ссылка на социальные сети с полными отзывами

**Дизайн**:
- Карточки в виде слайдера или сетки
- Кавычки для визуального оформления
- Светлый фон с тенями для карточек
- Автоматическая прокрутка отзывов (опционально)

### 6. Контакты (Contact Section)

**Назначение**: предоставление всей необходимой информации для связи

**Содержание**:
- Заголовок: "Контакты"
- Подзаголовок: "Свяжитесь со мной удобным способом"
- Контактная информация:
  - **Телефон**: с прямой ссылкой для звонка
  - **Адрес**: полный адрес салона в Сочи
  - **Часы работы**: график работы
  - **Email**: для вопросов и предложений
- Социальные сети:
  - Instagram
  - Telegram (основной канал)
  - WhatsApp
  - VK (опционально)
- Интерактивная карта (Яндекс.Карты или Google Maps) с меткой салона
- Форма обратной связи (опционально):
  - Имя
  - Телефон
  - Сообщение
  - Кнопка "Отправить"

**Дизайн**:
- Двухколоночная раскладка: контакты + карта
- Иконки для каждого способа связи
- Кликабельные ссылки для быстрой связи
- Тёмный или акцентный фон для выделения

### Footer (Подвал сайта)

**Содержание**:
- Копирайт "© 2024 Я Инна. Все права защищены"
- Краткая информация о салоне
- Дублирование ссылок на социальные сети
- Политика конфиденциальности (ссылка)
- Кнопка "Наверх" для быстрого возврата

**Дизайн**:
- Тёмный фон
- Минималистичный дизайн
- Компактное расположение информации

## Технический стек

### Frontend

- **Язык**: Rust (edition 2024)
- **Фреймворк**: Yew (CSR mode)
- **Компиляция**: WebAssembly (WASM)
- **CSS**: Bulma
- **Сборка**: Trunk

### Основные зависимости

- `yew` — UI фреймворк
- `wasm-bindgen` — связывание Rust и JavaScript
- `web-sys` — Web API bindings
- `serde` / `serde_yaml` — парсинг контента

### Инфраструктура

- **Хостинг**: статический хостинг (Cloudflare Pages, Netlify)
- **SEO**: meta-теги, Open Graph
- **Аналитика**: Яндекс.Метрика

## Архитектура

Проект построен как Single Page Application (SPA):

- **main.rs** — точка входа, инициализация приложения

### Структура контента

Контент сайта хранится в `static/content.yaml`:

- `info` — информация о салоне
- `services` — список услуг с ценами
- `contacts` — контактные данные

## Преимущества решения

- **Производительность**: WebAssembly обеспечивает быструю загрузку
- **SEO-оптимизация**: правильные мета-теги для локального поиска
- **Мобильная адаптивность**: полностью адаптивный дизайн
- **Поддерживаемость**: контент отделен от кода (YAML)
- **Современность**: использование Rust + WASM
- **Безопасность**: типобезопасность Rust
